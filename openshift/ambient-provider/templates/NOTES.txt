Verifying the Deployment
--------------------------------------------------------------------------------

Check all pods are running:
  oc get pods -n {{ .Values.namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

Check service health:
  oc exec -it $(oc get pod -l app.kubernetes.io/component=api -n {{ .Values.namespace }} -o name | head -1) -n {{ .Values.namespace }} -- curl localhost:{{ .Values.service.api.port }}/api/health

View logs:
  oc logs -f deployment/{{ include "ambient-provider.fullname" . }}-api -n {{ .Values.namespace }}

--------------------------------------------------------------------------------
Troubleshooting
--------------------------------------------------------------------------------

If pods are failing:
  oc describe pod <pod-name> -n {{ .Values.namespace }}
  oc logs <pod-name> -n {{ .Values.namespace }}

If API can't connect to NIMs:
  - Verify NVIDIA_API_KEY and (for cloud Riva) RIVA_FUNCTION_ID are set
  - Check network access to {{ .Values.nvidia.llmEndpoint }} and grpc.nvcf.nvidia.com:443
  - If you see "connection attempt timed out" to grpc.nvcf.nvidia.com, check cluster egress (firewall, proxy, NetworkPolicy) allows outbound HTTPS to *.nvcf.nvidia.com
